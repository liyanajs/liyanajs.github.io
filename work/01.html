GAME DASHBOARD

Full Stack Development / Changokushi Heroes

The "Changokushi Heroes" dashboard was born out of a necessity for clarity. The game team needed a way to visualize complex economy flow and player engagement metrics in real-time, moving away from static spreadsheets to a living, breathing interface.


This project became my proving ground for Full Stack development. I didn't just build the interface; I grew to engineer the data layer that powers it, defining the API structures that bridge our core game engine and the web.

ROLE Full Stack Developer
YEAR 2024
TECH STACK
TypeScript Next.js (API Routes) .NET Core Chart.js

The "Hybrid API" Architecture

The core game logic resides in a high-performance .NET environment. To bridge this with our web dashboard, we utilized Next.js API routes as a middleware layer.


Initially, I worked with endpoints established by senior engineers to build the UI. However, as the project scaled, I took ownership of the backend integration. I transitioned into designing my own API endpointsâ€”defining the response structures, aggregating data from multiple services, and collaborating with the team to finalize the JSON contracts. It was a pivotal period where I moved from simply consuming APIs to engineering them.

Visualizing the Economy

Data is only useful if it tells a story. I leveraged TypeScript to ensure strict type safety from the database all the way to the UI components.


The dashboard features dynamic charts that update as the game economy shifts, flagging anomalies like currency inflation or server load spikes instantly. This required writing custom API endpoints in Next.js to cache and calculate trends that didn't exist in the original game server.
